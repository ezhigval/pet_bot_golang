ЗЕЛЕНИАЛ – Безопасность и обработка данных
1. Все тяжелые вычисления на сервере

Принцип: LLM, аналитика, прогнозы, обработка больших данных — всё выполняется на бэкенде на мощных серверах.

Зачем:

Обеспечивает быстрый отклик приложений (мобильных и веб)

Не перегружает устройство пользователя

Позволяет использовать сложные модели AI и ML, без ограничения производительности на клиенте

Пример: пользователь задает вопрос “Почему я быстро устаю?” → данные уходят на сервер, AI обрабатывает персональные и обезличенные данные, формирует рекомендацию → результат быстро приходит в мобильное приложение или веб-клиент.

2. Персональные данные – шифрование и анонимизация

Две категории данных:

Персональные данные: возраст, здоровье, график дня, финансовая информация

Хранятся в отдельной базе данных

Шифруются персонально для каждого пользователя (ключ хранится у пользователя или в безопасном хранилище)

Доступ к этим данным имеет только пользователь и сервер для обработки конкретного запроса

Обезличенные данные: общие знания, инструкции, рекомендации, FAQ, советы по быту, обучение

Можно использовать для AI-запросов без риска раскрытия личной информации

Используются для обучения моделей и генерации рекомендаций в анонимной форме

Пример:

Вопрос: “Как оттереть жирное пятно на кухне?” → данные обезличенные, отправляем на сервер без персональной информации

Вопрос: “Почему Вася быстро устает?” → сервер собирает необходимые персональные параметры, анонимизирует их (например, вместо имени – уникальный ID), передает в AI-модель для ответа, возвращает результат пользователю

3. Безопасное взаимодействие клиент-сервер

Все соединения: TLS 1.3 (шифрование данных в пути)

Дополнительная защита:

HMAC / цифровая подпись запросов и ответов для предотвращения перехвата и модификации данных

Использование двухфакторной аутентификации (2FA) при доступе к чувствительным данным

Регулярное обновление сертификатов и ключей шифрования

Пример:

Любой запрос к API сначала проходит проверку подписи → сервер проверяет целостность и подлинность запроса → данные расшифровываются только после успешной проверки → AI генерирует ответ → результат шифруется и отправляется обратно клиенту

4. Архитектурное отражение безопасности
   +-------------------+         +--------------------+
   |  Мобильный/Веб    | <---->  |   API Gateway      |
   |  клиент           |         |  (TLS, HMAC)      |
   +-------------------+         +--------------------+
   |                          |
   v                          v
   +----------------+         +------------------+
   | Обезличенные   |         | Персональные     |
   | данные         |         | данные (шифр.)  |
   | (общие)        |         +------------------+
   +----------------+                   |
   |                               v
   v                        +------------------+
   +----------------+             |  AI / ML сервера |
   | AI / ML модели |             |  обработка       |
   +----------------+             +------------------+


Обезличенные данные: используются для рекомендаций и обучения AI без риска утечки

Персональные данные: шифруются и передаются только при необходимости, без раскрытия лишней информации

Все вычисления: на сервере → клиент получает только результат

5. Дополнительные меры

Логи, мониторинг, анонимизация аналитики

Регулярные аудиты безопасности и GDPR/RF-compliance проверки

Минимизация хранения данных на клиенте (только кэш, локальное шифрование для офлайн-доступа)

Обучение пользователей принципам безопасного использования приложения

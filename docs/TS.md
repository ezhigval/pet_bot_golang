Техническая спецификация — проект ЗЕЛЕНИАЛ (SAAS)
1. Архитектура системы
   Микросервисы

User Service — регистрация, авторизация, 2FA, управление профилем

Finance Service — управление доходами/расходами, интеграции с банками, прогнозы

Calendar/Tasks Service — расписания, задачи, напоминания, события

Health & Emotional Intelligence Service — трекер привычек, стресс, медитации, физическая активность

Learning Service — трекер навыков, обучение, библиотека контента

Home/IoT Service — интеграция с умным домом, напоминания по бытовым задачам

Transport & Assets Service — контроль автомобиля, маршруты, платежи по ЖКУ

Projects/Business Service — управление проектами и бизнесом, анализ рынка, расчеты

Social/Events Service — встречи, рекомендации по общению, мемы, актуальные события

AI/Assistant Service — генерация рекомендаций, обучение модели, чат-ассистент

Базы данных

PostgreSQL — основная реляционная база для микросервисов

Redis — кэширование, быстрый доступ к сессиям, очередям задач

TimescaleDB (опционально) — временные ряды для трекеров и статистики

Отдельная база для персональных данных — шифрование на уровне записи, доступ только у пользователя

Брокеры сообщений

Kafka — асинхронные события между микросервисами

Коммуникация

REST API + gRPC между сервисами и фронтендом

Форматы данных: JSON / Protobuf

2. Потоки данных

Пользователь отправляет запрос → API Gateway → нужный микросервис

AI/Assistant получает обезличенные или зашифрованные данные → обрабатывает → возвращает результат

Критичные данные шифруются на уровне сервиса и хранятся отдельно

Примеры сценариев:

Finance: пользователь планирует расходы → сервис собирает данные о доходах и регулярных платежах → прогноз, уведомления

Health: пользователь устал → фиксируется стресс → AI предлагает медитацию и корректирует расписание

Learning: пользователь хочет изучить Go → Learning Service планирует уроки с учетом времени и нагрузки

5. CI/CD пайплайны

Бэкенд (Go): Build → Test → Lint → Docker → Deploy

Веб-фронтенд (React): Build → Test → Lint → Deploy

Мобильные приложения: сборка → тесты → Deploy в TestFlight/Play Store Beta

Инструменты: GitHub Actions / Drone CI / Jenkins

6. Среда разработки и контейнеризация

Docker: каждый микросервис в отдельном контейнере

Docker Compose для локальной разработки

VSCode + плагины Go / Kotlin / Swift

Сервисы: PostgreSQL, Redis, Kafka через контейнеры

Локальный AI/ML сервер через Docker для разработки

7. Минимальная облачная среда для MVP

Облако: AWS / GCP / Яндекс.Cloud

Сервера: 2–3 бэкенд-узла (Load Balancer + Auto Scaling)

PostgreSQL + резервные копии

Kafka брокеры

CI/CD в облаке: автоматический деплой на тестовую среду

Мониторинг: Prometheus + Grafana

Безопасность: TLS, firewall, шифрование данных, VPN для админов

8. Принципы безопасности и конфиденциальности

Тяжелые вычисления выполняются на сервере, а не на устройстве пользователя

Персональные данные шифруются и хранятся отдельно

Взаимодействие клиент-сервер защищено от перехвата (TLS, дополнительное шифрование)

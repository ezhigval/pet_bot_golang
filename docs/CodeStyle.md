# Code Style Guidelines — ЗЕЛЕНИАЛ

Цель: единый стиль кода для всех микросервисов, фронтенда и мобильных приложений, читаемость, поддерживаемость и масштабируемость.

## Общие правила

- Язык: Go (Backend), JavaScript/TypeScript (Frontend), Kotlin/Swift (Mobile)
- Структура проекта должна быть модульной и однозначной.
- Использовать понятные имена функций, переменных и структур.
- Комментарии только там, где логика не очевидна.
- Логику не скрывать: код должен быть читаемым без «магии».

## Go (Backend)

- Файлы с маленьким количеством строк (~200–300 стр.).
- Структуры и интерфейсы именовать с заглавной буквы для экспортируемых сущностей.
- Функции и переменные: camelCase.
- Error handling обязателен.
- Использовать контексты (`context.Context`) для всех запросов к БД и внешним сервисам.
- Логирование через Zerolog.

## Frontend (React + Vite)

- Использовать функциональные компоненты + хуки.
- Стейт менеджмент через Context API или Redux Toolkit.
- Форматирование через Prettier.
- Все стили компонентные (CSS Modules или Tailwind).

## Mobile (Kotlin / Swift)

- Kotlin: follow Google Kotlin Style Guide.
- Swift: follow Swift API Design Guidelines.
- Все экраны разделять по функциональным модулям.

## Git

- Коммиты по Conventional Commits:
    - `feat:` — новая функциональность
    - `fix:` — исправление багов
    - `chore:` — служебные задачи
- Ветка `main` только для релизов.
- Все новые фичи в отдельных ветках `feature/название`.

## Документация

- Каждый сервис должен иметь README и структуру папок.
- Все схемы и диаграммы — в формате PNG или SVG в `docs`.

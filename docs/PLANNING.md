# Планирование и дорожная карта проекта ЗЕЛЕНИАЛ

Цель: структурировать процесс разработки от идеи до MVP и полной версии продукта, учитывая микросервисную архитектуру и монорепозиторий.

## 1. Этапы разработки

### Блок 1 — Проектирование
- Определение архитектуры микросервисов
- Продумывание бизнес-логики и основных сценариев
- Создание диаграмм и структуры папок/файлов
- Написание общей документации (README, TS, SAAS, CodeStyle)

### Блок 2 — User Service (Backend MVP)
- Регистрация и авторизация пользователей
- Управление профилем
- Привязка устройств, 2FA
- Безопасность и защита от многократных регистраций с одного устройства
- Настройка базы данных (PostgreSQL)
- Логирование и мониторинг через Zerolog

### Блок 3 — Frontend Web (MVP)
- React + Vite
- Вход и регистрация
- Просмотр профиля
- Экран `/health`
- Навигация по MVP функционалу

### Блок 4 — Mobile App (MVP)
- Kotlin / Swift
- Регистрация и вход
- Профиль пользователя
- Push-уведомления

### Блок 5 — Полный Backend
- Расширение User Service
- Добавление остальных микросервисов (финансы, здоровье, обучение, проекты)
- Настройка Kafka для коммуникации микросервисов
- Безопасность, защита данных, масштабируемость

### Блок 6 — Полный Frontend
- Web и Mobile
- Полный доступ ко всем блокам: здоровье, привычки, проекты, финансы, рефлексия
- Настройка уведомлений и аналитики

### Блок 7 — Первый релиз и запуск
- Тестирование на продакшн среде
- Оптимизация производительности и UX
- Маркетинговый запуск и привлечение первых пользователей
- Мониторинг безопасности и реакция на фидбек

## 2. Ключевые правила разработки
- Единый стиль кода (CodeStyle.md)
- Единая структура репозитория
- Ветка `main` — стабильная, `dev` — разработка
- Пуш и pull requests через GitHub
- Документация каждого микросервиса и API

## 3. Параллельная разработка
- Блоки 2, 3 и 4 могут разрабатываться параллельно после завершения проектирования
- Каждый микросервис и фронтенд модуль имеют свои поддиректории
- Вся команда работает в монорепозитории, позднее микросервисы могут быть вынесены на отдельные сервера

## 4. Контроль качества
- Code review для всех PR
- Юнит-тесты и интеграционные тесты
- CI/CD пайплайны для автоматической сборки и деплоя
- Проверка безопасности и нагрузочного тестирования
